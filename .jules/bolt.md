## 2024-05-24 - Lift constant config lookups and array initializations out of tight loops
**Learning:** Config lookups inside a hot path inner loop (like iterating through bounding boxes/slices in an image) take a toll, especially when dictionary methods are used `.get('key', default)`. If the configuration keys do not change during iterations, looking them up before the loop significantly decreases loop overhead. Additionally, initializing arrays on every fallback (like `np.zeros(h)`) costs allocation overhead and should be hoisted as fallback map variables.
**Action:** When working with image processing operations that loop over hundreds/thousands of slices/objects, review the body for any constant configuration lookups or redundant empty array initializations. Extract these to local variables defined *before* the inner loop begins.
